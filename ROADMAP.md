# Roadmap

slopless's near-term direction lives in two planning documents:

- [Promotion plan](.plans/2026-05-23-152521-promotion-master.md) - prioritized inventory of every directory, registry, marketplace, awesome-list, and editorial outlet to submit slopless to, with per-target acceptance bars and a wave-based schedule.
- [Repo excellence master](https://github.com/tartakovsky/kb/blob/main/.plans/2026-05-23-205345-slopless-repo-excellence-master.md) - what's shipped, what's next, and where slopless sits against the OpenSSF Best Practices baseline.

Issues are tracked at <https://github.com/seochecks-ai/slopless/issues>. Open epics are tagged with the `Epic:` prefix in their title.

## Near-term themes

- Distribution surfaces beyond npm: GitHub Marketplace action, VS Code + Open VSX extension, Homebrew tap, mise / AUR / Nixpkgs.
- Rules-only package split (`textlint-rule-preset-slopless`) for users who already have textlint configured.
- Wave 1 directory submissions to awesome-lists, dev directories, and the textlint Collection-of-rule wiki.
- OpenSSF Best Practices Silver tier.

## Long-term themes

- Output formats beyond JSON (human-readable, GitHub Actions annotations, SARIF).
- Auto-fix for deterministic rules (smart quotes, em-dashes, simplicity pairs).
- A scored sentence-level model for semantic-thinness and informativeness signals (tracked in the open `Scorer:` issues).
