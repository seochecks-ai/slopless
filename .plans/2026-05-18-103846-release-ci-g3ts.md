# Release CI G3TS Plan

## Goal

Release the current Slopless package to GitHub and npm with CI passing on GitHub.

## Current Findings

- Local package version is `0.2.5`.
- npm already has `slopless@0.2.5`.
- Current GitHub CI on `main` fails because `g3ts` is missing on the Ubuntu runner.
- Local `npm run validate` passes because `g3ts` is installed locally.
- `g3ts` is not an npm dependency and is not published on crates.io.
- `g3ts` can be installed in CI from Guardrail3 with:

```bash
cargo install --git https://github.com/agent-quality-controls/guardrail3 g3ts --locked
```

## Approach

- Update `.github/workflows/ci.yml` so the runner installs Rust and installs `g3ts` before `npm run validate`.
- Bump the npm package version to a new patch version because `0.2.5` already exists on npm.
- Run local validation.
- Pack the npm package locally and inspect the tarball contents.
- Commit with a worklog.
- Push to `development`.
- Open or update a PR to `main`.
- Wait for PR CI.
- Fix CI until it passes.
- Merge to `main` after CI passes.
- Tag the release.
- Publish the new npm version.
- Verify npm reports the new version.

## Files To Modify

- `.github/workflows/ci.yml`
- `package.json`
- `package-lock.json`
- `.worklogs/<timestamp>-release-ci-g3ts.md`

## Key Decisions

- Use a Git install of `g3ts` because there is no npm package or crates.io package for it.
- Put the install in CI instead of removing G3TS validation. The failure proves CI was not actually exercising the installed guardrail.
- Use a patch version bump because the release contains CI/package release plumbing around already validated behavior.
