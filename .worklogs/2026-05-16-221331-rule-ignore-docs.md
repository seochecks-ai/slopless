# Summary

Added documented support for ignoring specific Slopless rules with textlint comment directives.

The Slopless wrapper now ships and loads `textlint-filter-rule-comments` by default, so documented `textlint-disable` blocks work without user config.

# Decisions Made

- Used `textlint-filter-rule-comments` instead of inventing a Slopless-specific ignore syntax.
- Documented full rule IDs like `slopless/semantic-thinness` because that is what the wrapper emits in JSON.
- Shipped `slopless.textlintrc.json` with the npm package so the wrapper can enable the comments filter automatically.
- Bumped to `0.2.5` because package behavior and dependencies changed.
