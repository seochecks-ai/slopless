# Bundle size kill: swap @lunarisapp/readability for text-readability

## Summary

bundlephobia size dropped from 10.6 MB raw / 2.98 MB gzip → expected ~50 KB raw / ~16 KB gzip (a 200x reduction). Local install footprint dropped from ~21 MB (`@lunarisapp/hyphen` + `@lunarisapp/cmudict`) to ~180 KB (`text-readability` + `syllable` + `pluralize`).

## Change

- Replace `@lunarisapp/readability` with `text-readability` in `dependencies`.
- Three rule files: `coleman-liau.ts`, `flesch-kincaid.ts`, `gunning-fog.ts`. One line change each (import) plus removing the `new TextReadability({lang})` constructor call (text-readability exports a singleton).
- Add `src/types/text-readability.d.ts` shim — text-readability ships no types of its own.
- Re-approve 17 fixture goldens. Only `coleman-liau`/`flesch-kincaid`/`gunning-fog` messages changed; all other rule output is byte-for-byte identical. Verified by stripping readability messages from approved + received halves and diffing — zero non-readability differences across all 19 suites.

## Score divergence (expected, accepted)

The two libraries use different syllable counters. Coleman-Liau is letter/word/sentence arithmetic only, so scores match within ±0.13. Flesch Reading Ease and Gunning Fog use syllable counts:

- `@lunarisapp` uses cmudict (7 MB phonetic dictionary) → ground-truth syllables.
- `text-readability` uses the `syllable` package → heuristic counter.

On the same input the heuristic typically scores text as slightly "easier" (lower Gunning Fog, higher Flesch). The semantic direction is preserved — the existing thresholds (Coleman-Liau ≤12, Flesch ≥61, Gunning Fog ≤12) still fire on the same inputs in the fixture corpus.

## Revert: `declaration: false`

The earlier `declaration: true` change emitted `.d.ts` files into `dist/rules/*/`. `behavior-replay.sh` passes `--rulesdir dist/rules/X` to textlint, and textlint tries to load every file in that directory as a runtime rule — `.d.ts` files crash with `rule is not defined`. CI never caught this because CI runs `pnpm run validate` (lint/build/spell/typecov), not the fixture replay.

Reverted to `declaration: false`. Re-enabling consumer type declarations needs a proper fix: emit declarations to `dist-types/` via `declarationDir`, then add per-export `types` entries in `package.json` `exports`. Out of scope here; tracked as a follow-up.

## Not regressing

- CLI version reporting unchanged (package.json bumps will continue to flow through).
