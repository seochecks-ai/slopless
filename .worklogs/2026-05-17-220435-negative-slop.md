Summary

Added constrained negative slop coverage to the existing syntactic contrast rule path. The implementation extends `syntactic-patterns:negation-reframe` instead of creating a new family, and mirrors the accepted cases into corpus prose.

Decisions made

- Kept negative slop under `syntactic-patterns/contrast` because it is a negation contrast frame, not a separate rule family.
- Split the new matcher helpers into `negative-slop-frames.ts` so `negation-reframe-matcher.ts` stays inside the manifest line limit.
- Limited the new matching to stable frames:
  - `was no longer X. It was Y.`
  - `not a/an/the X. It was Y.`
  - `did not X. Same-subject Y.`
  - `didn't X. Instead, same-subject Y.`
- Rejected a broad generic negation detector because it would catch factual corrections and normal contrast.
- Accepted only the intended fixture output after checking that the new findings were expected and no no-hit findings appeared.
