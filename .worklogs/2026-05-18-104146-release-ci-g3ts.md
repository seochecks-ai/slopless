# Release CI G3TS

## Summary

Prepared the Slopless `0.2.6` release and fixed GitHub CI so it installs the local Guardrail3 TypeScript validator before running `npm run validate`.

## Decisions Made

- Installed `g3ts` in CI with `cargo install --git https://github.com/agent-quality-controls/guardrail3 g3ts --locked` because the package validator is a Rust binary in Guardrail3, not an npm package.
- Kept Slopless validation on `g3ts validate --path . --rules-only`; repo-level `validate-repo` remains blocked by the unresolved Guardrail3 hook-contract issues recorded in the sibling Guardrail3 plan.
- Bumped the npm version from `0.2.5` to `0.2.6` because `0.2.5` is already published.
- Fixed `verify-split-slopless.py` so release version bumps are valid. It now checks semver shape and package-lock consistency instead of treating `0.2.5` as permanent repo shape.
