# pnpm 11: explicitly allow unrs-resolver native build

## Summary

CI failed under pnpm 11 with `ERR_PNPM_IGNORED_BUILDS: unrs-resolver@1.11.1`. In pnpm 11, ignored build scripts are a hard error during `--frozen-lockfile` installs, not a warning. `unrs-resolver` (used transitively by `eslint-plugin-import-x`) builds a native binary on install and must be explicitly approved.

## Fix

Add `pnpm-workspace.yaml` at repo root with:

```yaml
allowBuilds:
  unrs-resolver: true
```

pnpm 11 also moved settings out of the `package.json` `pnpm` field — the new home for these settings is `pnpm-workspace.yaml` (which is no longer workspace-only; pnpm 11 reads it from any project). `package.json` `pnpm.onlyBuiltDependencies` is silently ignored with a warning.

Verified locally: `rm -rf node_modules && pnpm install --frozen-lockfile` completes cleanly; `pnpm run build` produces output.

## Note

The previous worklog `2026-05-23-183303-pnpm-action-version-fix.md` claimed CI would turn green after dropping `version:` from `pnpm/action-setup@v4`. That fix was necessary but not sufficient — the pnpm 11 ignored-builds error surfaced after the action-setup error was resolved. This is one of those onion situations where each layer reveals the next.
